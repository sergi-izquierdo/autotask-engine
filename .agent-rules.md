# Agent Rules - AutoTask Engine

## Environment Bootstrap (CRITICAL)
- Before running ANY build/dev/test command, ALWAYS run npm install first
- If a CLI tool is not found, install dependencies and retry before escalating

## Workflow
- Never push directly to main, always work on your assigned branch
- Commit atomically per task with clear messages
- Run tests before opening/updating PRs
- Use TypeScript strict mode everywhere

## Code Quality
- Follow existing patterns in the codebase
- Add tests for new functionality
- Do not refactor code outside your assigned scope
- Use Zod for all runtime validation
- Handle errors gracefully with typed error responses

## Self-Healing Patterns
- Missing dependency: install it, then retry
- Build failure: read error output, attempt fix, retry (max 2 attempts)
- Port conflict: try next port
- Permission error: check file ownership, do NOT use sudo

## Architecture
- Monorepo with packages/core, packages/api, packages/web, packages/scheduler
- Core: pure TypeScript, no framework dependencies
- API: Hono on Bun for speed
- Web: Next.js 15 with App Router
- Scheduler: Bun-based cron runner
- Database: SQLite via Drizzle ORM
- Queue: BullMQ with Redis for task execution
